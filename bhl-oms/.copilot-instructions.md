# BHL OMS-TMS-WMS — Copilot Instructions

## Project Context

Hệ thống vận hành Bia Hạ Long: Đặt hàng → Điều vận → Xuất kho → Giao hàng → Đối soát.
~1,000 đơn/ngày, ~800 NPP, ~70 xe, ~50 users, ~30 SKU.

## Architecture

3-layer per module: `Handler → Service → Repository`

```
bhl-oms/
├── cmd/server/main.go          ← Entry point, DI wiring
├── internal/
│   ├── domain/models.go        ← All shared structs
│   ├── oms/                    ← handler.go, service.go, repository.go
│   ├── tms/                    ← same pattern
│   ├── wms/                    ← same pattern
│   ├── auth/                   ← JWT RS256
│   ├── gps/                    ← WebSocket hub
│   └── middleware/             ← auth.go
├── pkg/
│   ├── db/                     ← pgxpool connection
│   └── response/               ← Standard API response helpers
├── migrations/                 ← SQL files
├── vrp-solver/                 ← Python service
└── web/                        ← Next.js frontend
```

## Critical Rules

1. **pgx v5:** Always cast `::text` for enum and date columns in SELECT
2. **Money:** Use `NUMERIC(15,2)` in DB, never float64 in Go
3. **Timezone:** UTC in DB, convert `Asia/Ho_Chi_Minh` in app
4. **Existing code is source of truth** — specs in `docs/specs/` are targets for NEW code
5. **Response format:** Use `pkg/response/` — `response.OK()`, `response.Err()`, etc.
6. **Error codes:** Reference `docs/specs/ERROR_CATALOGUE.md`
7. **State machines:** Reference `docs/specs/STATE_MACHINES.md`
8. **Business rules:** Reference `docs/specs/BUSINESS_RULES.md`
